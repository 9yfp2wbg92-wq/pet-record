# 宠记 — 宠物生活记录与健康管理

> 朝夕相伴 · 岁岁留痕

一个帮助宠物主人记录日常、追踪健康、沉淀回忆的轻量级工具。从"记了什么"到"什么时候该做什么"，让养宠这件事变得有条理又不失温度。

📱 **在线体验**：[pet-record.vercel.app](https://pet-record.vercel.app)

---

## 一、这个产品解决什么问题

养宠物的人都有类似的经历：疫苗本不知道塞哪儿了、上次驱虫是三个月还是四个月前想不起来、换季时宠物突然软便但不确定是不是因为换了粮。日常碎片散落在相册、备忘录、微信聊天记录里，关键时刻拼不出一条完整的健康轨迹。

**宠记**用两个核心机制来解决这个问题：

1. **动态 + 里程碑双轨记录** — 日常随手发一条动态（像发朋友圈一样简单），AI 自动从中识别出疫苗、驱虫、体重等健康事件，沉淀为可追溯的时间轴档案；不想用 AI 也可以手动添加
2. **护理倒计时看板** — 把分散的疫苗到期日、驱虫间隔、体重变化集中在同一页，用进度条直观展示"还剩几天该做某事了"，不再靠脑子记

一句话概括：**让日常记录有地方去，让健康数据回头看得到。**

---

## 二、目标用户

养宠的中国家庭，尤其是——

- **新手养宠人**：第一次养猫狗，对疫苗驱虫流程不熟悉，需要一个"引导式"的记录工具
- **多宠家庭**：家里猫狗双全或多只同品种，需要清晰区分每只宠物的档案和健康记录
- **宠物的"主理人"**：负责带宠物看病、记录健康数据的那个人（通常是家庭里最操心的一位）

使用场景多是碎片时间：遛狗回来看一眼驱虫倒计时、带猫体检后记下医生说的注意事项、换粮期记录排便情况观察是否适应。

## 三、核心功能 & 产品决策

### 功能一览

| 模块 | 做什么 | 
|------|--------|
| 📝 动态记录 | 文字+图片发布日常，AI 自动识别健康事件并归档 |
| 📅 大事年表 | 按月分组的时间轴，支持按事件类型筛选，倒序展示 |
| 🤖 AI 看板 | 30 天健康报告 + 护理倒计时（驱虫/疫苗/洗澡） |
| 🐱 多宠物档案 | 每只独立档案，独立配色，支持多宠切换 |
| 👨‍👩‍👧‍👦 家庭共享 | 6 位邀请码，家人可共同记录同一只宠物 |

### 几个关键产品决策

**1. 为什么 AI 识别做在客户端而不是云端？**

市面上有些宠物 App 把 AI 做成需要登录+联网的云端服务。我选择在客户端用规则引擎做本地识别（关键词匹配 + 正则）。理由是：养宠场景的输入高度模式化（"今天打了疫苗""体重 4.5kg"），规则覆盖率高；不需要后端，用户隐私完全在本地；零延迟，发布即识别。如果未来用户量上去，再考虑接云端 NLP 模型做更智能的语义理解。

**2. 为什么先做"大事年表"而不是做"社交 feed"？**

社交功能（点赞、评论）是有的，但产品主路径是记录→归档→看板，不是刷 feed。早期宠物类产品的核心留存来自"我记的东西对我和宠物有用"，而不是"有多少人给我点赞"。所以投入更多精力在时间轴的可读性（按月分组、分类筛选、宠物颜色区分）和看板的数据呈现上。

**3. 为什么多宠物用颜色区分而不是纯文字？**

多宠家庭的痛点之一是快速区分"这条记录是哪只的"。给每只宠物分配一个专属色（玫瑰/天蓝/翠绿/紫罗兰/琥珀/青绿），在时间轴、动态卡片、看板切换中保持一致——用户扫一眼颜色就知道是谁，不需要读文字。这个设计来自实际观察：很多宠物 App 用文字标签区分，导致滚动时辨识效率低。

**4. 为什么不做注册登录系统？**

当前版本用 localStorage 做本地存储，用户打开即用，不需要注册账号。这在产品早期有两个好处：降低试用门槛（不需要手机号/邮箱），打开链接就能直接体验完整 Demo 数据。后续考虑接 Supabase 做可选的云端同步，但本地模式作为"离线可用"的底线会一直保留。

## 四、技术实现

React + TypeScript + Vite 构建，Zustand 管理状态，Tailwind CSS 做样式，数据存储在浏览器 localStorage。AI 识别通过前端正则 + 关键词匹配实现，无需后端服务。

## 六、如果继续做，我会优先加什么

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | 云端同步（Supabase） | 目前数据存在本地，换设备或清缓存会丢失。先做宠物档案模块的云端同步作为 MVP |
| P1 | 数据导出（PDF 健康报告） | 用户带宠物看病时可以把疫苗史、体重曲线导出给兽医看 — 从"给自己看的工具"变成"医生也能用的参考" |
| P1 | 提醒推送 | 现在需要打开 App 才能看到倒计时。接 Web Push Notification，驱虫/疫苗到期前 3 天弹通知 |
| P2 | 体重曲线可视化 | 目前大事年表里体重是单独记录，做成折线图更直观判断趋势 |
| P2 | 宠物品种数据库 | 添加宠物时从预置品种库选择，自动带出该品种的标准体重范围，AI 看板可以对比判断偏瘦/偏胖 |
| P3 | 社区功能 | 轻量级的"同品种家长群"或话题圈子，但前提是个人记录功能成熟后再加社交层，否则会稀释核心价值 |
