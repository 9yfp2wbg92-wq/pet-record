```
┌─────────────────────────────────────────────────────────────────────────┐
│                        宠物记录应用架构图                                │
├─────────────────────────────────────────────────────────────────────────┤

│   │                        用户界面层                                │  │

│   │  │  Home   │  │Timeline │  │Insights │  │Profile │  │Modal  │ │  │

│   └───────┼────────────┼────────────┼────────────┼────────────┼─────┘  │
│           ▼            ▼            ▼            ▼            ▼        │
│   ┌──────────────────────────────────────────────────────────────────┐  │
│   │                        状态管理层                                │  │
│   │                    usePetStore (Zustand)                        │  │
│   │  currentUser | currentPetId | pets | posts | milestones | users │  │
│   └──────────────────────────────────────────────────────────────────┘  │
│                                   │                                     │

│   ┌──────────────────────────────────────────────────────────────────┐  │

│   │                   localStorage 操作封装                          │  │

```

### 1.2 核心数据模型

| 实体 | 核心字段 | 说明 |
|------|----------|------|
| **User** | id, name, avatar, createdAt | 用户信息 |
| **Pet** | id, name, category, gender, birthday, homeDate, avatar, ownerUserId | 宠物信息 |
| **Post** | id, petId, content, media, authorUserId, comments, likes | 动态内容 |
| **Milestone** | id, petId, type, title, description, date, metrics, isAI | 里程碑事件 |
| **Comment** | id, postId, content, authorUserId, createdAt | 评论 |
| **Like** | postId, userId, userName, createdAt | 点赞记录 |

## 2. 用户注册与登录流程

### 2.1 首次注册流程

```
用户进入应用
    │
    ▼
检测是否有已登录用户(localStorage)
    │
    ├─ 有 → 加载用户数据 → 进入首页
    │
    └─ 无 → 显示注册页面
            │
            ▼
        输入昵称
            │
            ▼
        点击"开始记录"
            │
            ▼
    创建用户(User)
            │
            ▼
    保存到localStorage
            │
            ▼
    设置currentUser
            │
            ▼
    进入首页(提示添加宠物)
```

**关键代码位置**：
- `/workspace/src/pages/Profile.tsx` - 注册页面逻辑
- `/workspace/src/hooks/usePetStore.ts` - `registerUser` 方法

### 2.2 用户切换流程

```
进入"我的"页面
    │
    ▼
点击设置按钮(右上角)
    │
    ▼
选择"切换用户"
    │
    ▼
显示用户列表
    │
    ▼
选择目标用户
    │
    ▼
调用switchUser(userId)
            │
            ▼
重新加载该用户的所有数据(pets/posts/milestones)
            │
            ▼
更新currentUser和currentPetId
            │
            ▼
刷新所有页面数据
```

## 3. 宠物管理流程

### 3.1 添加宠物流程

```
进入"我的"页面
    │
    ▼
点击"+ 添加宝贝"按钮
    │
    ▼
显示添加宠物表单
            │
            ├─ 输入宠物名称
            ├─ 选择宠物种类(10种选项 + 其他宝贝自定义)
            ├─ 选择性别(雄性/雌性)
            ├─ 设置生日(准确/估计/未知)
            ├─ 设置到家日期
            └─ 上传头像(可选)
                    │
                    ▼
                点击"保存"
                    │
                    ▼
            生成邀请码
                    │
                    ▼
            创建Pet对象
                    │
                    ▼
            保存到localStorage
                    │
                    ▼
            更新store.pets
                    │
                    ▼
            返回资料卡页面
```

**宠物种类选项**：
| 选项 | emoji | 说明 |
|------|-------|------|
| 猫咪 | 🐱 | 猫科动物 |
| 狗狗 | 🐶 | 犬科动物 |
| 鸟类 | 🐦 | 鸟类宠物 |
| 兔子 | 🐰 | 兔类 |
| 鱼类 | 🐟 | 观赏鱼 |
| 鼠鼠 | 🐹 | 仓鼠、鼠类、豚鼠、龙猫 |
| 爬宠 | 🦎 | 爬行动物 |
| 貂 | 🦦 | 宠物貂 |
| 蜜袋鼯 | 🐨 | 蜜袋鼯 |
| 刺猬 | 🦔 | 宠物刺猬 |

### 3.2 编辑宠物资料流程

```
进入宠物资料卡
    │
    ▼
点击资料卡任意位置
    │
    ▼
显示编辑表单(与添加表单相同)
            │
            ▼
        修改信息
            │
            ▼
        点击"保存"
            │
            ▼
    更新Pet对象
            │
            ▼
    保存到localStorage
            │
            ▼
    更新store.pets
            │
            ▼
    刷新资料卡显示
            │
            ▼
    更新相关里程碑(如生日、到家日期)
```

**性别显示规则**：
- 雄性(弟弟)：蓝色标签 `bg-blue-100 text-blue-600`
- 雌性(妹妹)：粉色标签 `bg-pink-100 text-pink-600`

### 3.3 邀请家庭成员流程

```
进入宠物资料卡
    │
    ▼
点击"邀请家人"按钮
    │
    ▼
生成邀请码(6位)
            │
            ▼
        复制邀请码
            │
            ▼
        分享给家人
            │
            ▼
    家人使用邀请码加入
            │
            ▼
        更新pet.sharedUserIds
            │
            ▼
        家人可查看该宠物
```

## 4. 动态发布与查看流程

### 4.1 发布动态流程

```
进入记录页面
    │
    ▼
选择宠物(顶部筛选器)
    │
    ▼
点击输入框
    │
    ▼
输入文字内容
            │
            ▼
        [可选]点击相册图标
                │
                ▼
            选择图片
                    │
                    ▼
                上传图片
                        │
                        ▼
                    AI识别健康事件
                            │
                            ▼
                        显示识别结果弹窗
                                │
                                ▼
                            勾选要归档的事件
                                    │
                                    ▼
                                设置提醒间隔(洗澡/驱虫)
                                        │
                                        ▼
                                    点击"确认归档"
                                            │
                                            ▼
                                        添加到大事年表
                                                │
                                                ▼
                                            点击"发布"
                                                    │
                                                    ▼
                                                创建Post对象
                                                        │
                                                        ▼
                                                    保存到localStorage
                                                            │
                                                            ▼
                                                        更新store.posts
                                                                │
                                                                ▼
                                                            更新AI看板
```

**AI识别事件类型**：
- 💉 疫苗
- 🐛 驱虫
- ⚖️ 体重记录
- 🛁 洗澡
- 🩺 就医
- ⚠️ 异常

### 4.2 查看动态流程

```
进入记录页面
    │
    ▼

    │

                                ▼
                            点击心形图标
                                    │
                                    ▼
                                更新点赞状态(立即响应)
                                        │
                                        ▼
                                    保存到store和localStorage
                                                │
                                                ▼
                                            [可选]评论动态
                                                    │
                                                    ▼

                                                        ▼

                                                            ▼
                                                        创建Comment对象
                                                                │
                                                                ▼

                                                                    │
                                                                    ▼
```

## 5. 大事年表流程

### 5.1 查看大事年表流程

```
进入大事年表页面
    │
    ▼
选择宠物筛选(全部/特定宠物)
    │
    ▼
选择类型筛选(全部/疫苗/驱虫/体重/就医/异常)
    │
    ▼
加载并显示时间轴
            │
            ▼
        按时间倒序显示事件
                │
                ├─ 时间轴节点(图标+日期)
                └─ 事件卡片(类型标签+内容+操作按钮)
                        │
                        ▼
            [可选]编辑事件
                    │
                    ▼
                弹出编辑弹窗
                        │
                        ▼
                    修改信息并保存
                            │
                            ▼
                        更新Milestone对象
                                │
                                ▼
                            保存到localStorage
                                    │
                                    ▼
                                更新store.milestones
                                            │
                                            ▼
                                        [可选]删除事件
                                                │
                                                ▼
                                            确认删除
                                                    │
                                                    ▼
                                                从store和localStorage删除
```

**时间轴显示规则**：
- 显示完整年份（YYYY年MM月DD日）
- 紧凑布局，减少左侧空白
- 按日期倒序排列（最新的在上）
- AI归档的事件优先显示

### 5.2 添加里程碑流程

```
进入大事年表页面
    │
    ▼
点击"+"按钮
    │
    ▼
显示添加事件表单
            │
            ├─ 选择事件类型
            ├─ 输入标题
            ├─ 输入描述(可选)
            ├─ 选择日期
            └─ 输入指标(如体重)
                    │
                    ▼
                点击"保存"
                    │
                    ▼
            创建Milestone对象
                    │
                    ▼
            关联到当前选中的宠物
                    │
                    ▼
            保存到localStorage
                    │
                    ▼
            更新store.milestones
                    │
                    ▼
            更新AI看板健康报告
```

## 6. AI看板流程

### 6.1 健康报告生成流程

```
进入AI看板页面
    │
    ▼
选择宠物(默认显示第一个宠物)
    │
    ▼
统计健康相关事件(排除到家/生日)
            │
            ▼
        判断事件数量
                │
                ├─ 0条 → 显示"期待第一条记录"提示
                │
                ├─ 1条 → 显示"期待第二条记录"提示
                │
                └─ ≥2条 → 自动生成健康报告
                            │
                            ▼
                        分析健康趋势
                                │
                                ▼
                            生成健康建议
                                    │
                                    ▼
                                显示健康评分
```

**健康报告生成规则**：
- 排除事件类型：到家(home)、纪念日(anniversary)
- 纳入事件类型：疫苗、驱虫、体重、就医、异常、洗澡
- 实时更新：当动态或里程碑更新时自动重新生成

### 6.2 健康报告内容

```
健康报告
    │
    ├─ 健康评分(0-100)
    │
    ├─ 健康趋势分析
    │       │
    │       ├─ 体重变化趋势
    │       ├─ 疫苗接种情况
    │       └─ 驱虫频率分析
    │
    ├─ 健康建议
    │       │
    │       ├─ 待完成事项提醒
    │       ├─ 异常情况预警
    │       └─ 日常护理建议
    │
    └─ 历史记录摘要
            │
            ├─ 总事件数
            ├─ 最近事件
            └─ 平均事件间隔
```

## 7. 跨页面宠物选择同步机制

### 7.1 状态同步架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    宠物选择同步机制                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │   记录页面    │    │   大事年表    │    │   AI看板     │      │
│  │   (Home)     │    │  (Timeline)  │    │ (AIInsights) │      │
│  │   选择宠物    │    │   选择宠物    │    │   选择宠物    │      │
│  └──────┬───────┘    └──────┬───────┘    └──────┬───────┘      │
│         │                   │                   │               │
│         └───────────────────┼───────────────────┘               │
│                             ▼                                   │
│                    ┌──────────────┐                             │
│                    │  store       │                             │
│                    │ currentPetId │                             │
│                    └──────┬───────┘                             │
│                           │                                     │
│                           ▼                                     │
│                    localStorage                                  │
│                   pet_tracker_current_pet_id                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 默认显示规则

| 页面 | 默认显示 | 选择特定宠物后 |
|------|----------|----------------|
| 记录页 | 全部宠物动态 | 仅该宠物动态 |
| 大事年表 | 全部宠物事件 | 仅该宠物事件 |
| AI看板 | 第一个宠物 | 该宠物报告 |

### 7.3 同步流程

```
用户在任意页面选择宠物
    │
    ▼
调用setCurrentPetId(petId)
            │
            ▼
更新store.currentPetId
            │
            ▼
更新store.currentPet
            │
            ▼
保存到localStorage
            │
            ▼
所有监听该状态的组件自动更新
                    │
                    ▼
记录页：过滤显示该宠物动态
                    │
                    ▼
大事年表：过滤显示该宠物事件
                    │
                    ▼
AI看板：显示该宠物健康报告
```

## 8. 数据持久化机制

### 8.1 localStorage存储结构

| 存储键 | 数据类型 | 说明 |
|--------|----------|------|
| pet_tracker_users | User[] | 所有用户列表 |
| pet_tracker_pets | Pet[] | 所有宠物列表 |
| pet_tracker_posts | Post[] | 所有动态列表 |
| pet_tracker_milestones | Milestone[] | 所有里程碑列表 |
| pet_tracker_memes | Meme[] | 表情包列表 |
| pet_tracker_current_user_id | string | 当前登录用户ID |
| pet_tracker_current_pet_id | string | 当前选中宠物ID |

### 8.2 数据加载流程

```
应用启动
    │
    ▼
调用loadData()
            │
            ▼
读取currentUserId
            │
            ▼
获取对应用户的pets
            │
            ▼
根据petIds获取posts
            │
            ▼
根据petIds获取milestones
            │
            ▼
读取currentPetId
            │
            ▼
验证currentPetId有效性
            │
            ▼
设置store状态
            │
            ▼
页面组件重新渲染
```

### 8.3 数据更新流程

```
用户执行操作(添加/修改/删除)
    │
    ▼
调用store方法
            │
            ▼
更新store状态
            │
            ▼
调用storage方法
            │
            ▼
保存到localStorage
            │
            ▼
组件自动重新渲染
```
